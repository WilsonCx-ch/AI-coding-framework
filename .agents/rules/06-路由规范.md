# 路由规范（NON-NEGOTIABLE）

## 路由结构

- 路由目录在 `src/routes` 目录下
- 每个路由页面单独一个目录，使用 `kebab-case` 命名
- 路由之下必须包含 `Page.tsx`、`Loader.tsx` 和 `index.module.scss` 三个文件
- Loader 负责懒加载，Page 是主页面
- 如有子路由或专属组件，可在路由目录下添加 `routes/` 或 `components/` 目录

如需查看创建路由的完整示例与落地步骤，请使用技能文件：

- `.agents/skills/create-route/SKILL.md`

## 路由与菜单规范（NON-NEGOTIABLE）

- **禁止多处维护路由**：全局唯一的路由配置文件集中管理，禁止在其他地方写路由配置
- **Loader 职责收敛**：每个路由目录下的 `Loader.tsx` 只负责懒加载对应的 `Page` 组件，不嵌套 `<Routes>` / `<Route>`
- **避免重复 lazy 加载**：同一个页面/Loader 组件只允许在一个地方被 lazy 引入
